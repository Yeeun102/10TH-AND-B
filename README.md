## 🌳 branch 규칙
1. 본인의 닉네임으로 새 브랜치를 만들어 주세요.
2. 이 레포지토리를 본인 계정으로 Fork 합니다.
3. Fork한 레포지토리에 미션 결과물을 올립니다.
4. 본인 레포지토리에서 SSWU-UMC 레포지토리의 닉네임 브랜치로 Pull Request(PR)를 보내면 끝!

## 🍀 PR 규칙
- 제목 형식
    - [WeekN] 닉네임/이름 미션 제출
      
## 🔖 커밋 컨벤션
1. 커밋에 해당하는 커밋 메시지 쓰기 (예: [berr] feat: 1주차 과제 완료 )


| Message  | 설명                                              |
| :------: | :------------------------------------------------ |
|   feat   | 새로운 기능 추가                            |
|  fix   | 버그 수정 |
|  refactor  | 코드 리펙토링 |
| comment | 주석 추가 및 변경                                     |
|  rename   | 파일 혹은 폴더명 수정                |
|  remove   | 파일 혹은 폴더 삭제                |
|  chore   | 기타 변경사항                |
